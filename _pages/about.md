## Introduce
안녕하세요. 5년 차 백엔드 개발자 김재녕입니다.  
저는 ***Java, Spring Boot, AWS***를 주축으로 학습, 훈련하며 개발을 해왔습니다.  
주로 웹 서비스를 개발 및 운영해왔으며  
클래스, DB 설계부터 인프라 구축, 개발, 테스트, 배포, 운영까지의 ***모든 프로세스를 직접 경험***해보았습니다.  
이로 인해 프로젝트를 주도적으로 해결해 나갈 수 있는 역량을 기를 수 있었습니다.

새로운 기술보다는 ***사용자의 편의성을 위한 서비스를 추구***합니다.  
따라서 유연하고 안정성, 확장성 있는 리액티브 시스템 구축을 위한 ***인프라, 아키텍처가 주요 관심사***이며,  
빠르고 안전한 기능 반영과 버그 수정을 위한 애자일 개발 프로세스에 관심이 많습니다.

배우는 것을 좋아하여 그동안 10개 이상의 오프라인 강의를 수강한 경험이 있고  
학습 내용을 꾸준히 깃허브에 업로드하고 있어 현재 40여 개의 저장소가 있습니다.  
이를 밑거름으로 실제 프로젝트에 다양한 기술을 적용해보며 유연한 설계를 구축할 수 있었습니다.

언젠가 힘들고 어려운 그리고 지친 사람들을 위한 서비스를 만들고 싶습니다.

### ***AMP 프로젝트***
ADOP (2018.09 ~ 2019.01) 5개월간
> ***설명***
> * AMP(Accelerated Mobile Pages) 페이지 변환 모듈 개발 및 AMP 페이지 퍼블리싱 서버 운영
> 
> ***담당 업무***
> * 고객사(매체)가 제공하는 RSS를 AMP 데이터로 변환하는 모듈 구현
> * AMP 페이지의 SEO(Search Engine Optimization)를 위해 구글 SEO 가이드 확인 및 적용
> * 팀원이 개발한 AMP 페이지 퍼블리싱 서버(PHP, CI) 운영
> 
> ***개발 환경***
> * Java(8), Jsoup, MyBatis, Gradle, AWS(EC2, RDS)
> 
> ***문제 해결 사례***
> * 해외 매체들은 기업 내 기술력의 부재로 통일된 규격의 RSS를 제공하지 못함
> * 해외 매체만을 위한 매체별 사이트 기사 페이지에서 HTML을 직접 스크래핑하여 AMP 데이터로 파싱, 저장하는 별도 모듈 구현
> * 스크래핑, DB 입출력은 동일하나 데이터 파싱 과정이 매체마다 조금씩 달라 분기처리가 많아짐
>   * HTML이 정형화된 구조가 아니라 매체마다 각기 다른 구조로 렌더링 되어 있기 때문에 매체별로 파싱 작업이 다름
>   * 초창기 간단한 로직으로 인해 SOLID 원칙 등 객체 지향적 설계가 이루어지지 않았음
> * 매체별 파싱 작업이 다양해짐에 따라 수정 시 다른 매체의 파싱 작업이 영향을 받는 경우 발생
>   * 단순 ***테스트 코드 작성만으로는*** 파싱 작업의 무결성과 추출된 ***데이터가 유효한지 판단이 어려움***
> * 매체별 파싱 과정 작업을 ***Strategy 패턴을 적용하여 책임을 분담***하여 특정 매체 파싱 작업 수정 시 다른 매체에 영향이 없도록 처리
> 
> ***배운 점***
> * 비록 간단한 모듈일지라도 ***객체 지향적 설계***와 구현을 통해 책임과 역할을 분담하는 것이 얼마나 중요한지 깨닫게 되었습니다.
> * 클래스 아키텍처가 복잡하지 않았음에도 꽤 시간이 소요되는 작업이었습니다.
> * 설계 단계에서 유연한 설계에 대해 한 번 더 고민하는 계기가 되었습니다.
