**FHIR 데이터 ETL 파이프라인 구축** (2022.05~2022.10) 6개월

고객사인 네이버의 헬스케어 데이터 활용 목적으로 자사 EMR 서비스의 데이터를 네이버 FHIR(의료 데이터 국제 기술 표준) 서버에 전송하는 데이터 ETL(Extract-Transform-Load) 파이프라인 구축  
EMR API 서버에서 데이터 CRUD 이벤트 발생 > 발생한 서버에서 메시지 큐(Kafka)로 전송 > 컨슈머 모듈에서 메시지 컨슘 > [Extract] 컨슘한 메시지의 담긴 PK를 통해 OLTP용 RDB에서 조회 > [Transform] 조회한 데이터를 FHIR 포맷으로 변환 > [Load] 네이버 서버로 전송

* 자사 EMR API 서버에서 발생하는 데이터를 제로페이로드 포맷 메시지로 카프카 브로커에게 전송하는 기능 구현  
  데이터 무결성을 위해 메시지 큐를 도입, exactly once 옵션 설정  
  엔티티 모듈에서 JPA의 EntityListener를 활용해 데이터의 CRUD 상태를 감지하여 메시지 전송 처리  
  메시지 생성 순서에 영향받지 않기 위해 제로페이로드 방식을 활용, 컨슈머에게 데이터 무결성 책임 위임
* 카프카 브로커로부터 메시지를 컨슘하여 데이터 조회, 변환, 전송하는 컨슈머 모듈 구현  
  컨슘한 메시지의 PK를 통해 데이터를 다시 조회함으로써 데이터 무결성 달성  
  컬럼 각각의 데이터 타입을 맞춰주기 위해 HAPI FHIR 프레임워크를 활용해 자사 데이터를 FHIR 포맷으로 변환하는 기능 구현
* 엔티티 모듈을 의존하는 하위 모듈에 필요없는 카프카 의존성 제거  
  메시지 전송 처리 로직을 하위 API 서버 모듈로 이동, 엔티티 모듈에는 인터페이스화하여 모듈 리팩터링
* 컨슈머 모듈 Jenkins 빌드 파이프라인, ArgoCD 설정
