* 데이터 무결성을 위해 카프카 도입, 기존 서비스에 JPA @EntityListener를 사용하여 데이터 입출력 시 제로 페이로드 포맷으로 카프카 이벤트 생성 기능 추가, 데이터 최신화를 컨슈머에게 위임

* 카프카 브로커로부터 데이터를 읽어 FHIR 프레임워크를 활용해 FHIR 포맷으로 변경하여 네이버 FHIR 서버로 전송하는 카프카 컨슈머 모듈 구현 후 데이터 무결성을 위해 exactly once 옵션 설정
* 컨슘이 안되는 기존 데이터 일괄 전송 시 약 1만 건에 달하는 양으로 인해 발생하는 DB 커넥션 타임아웃을 컬렉션의 parallel(CommonThreadPool)과 스프링 @Async 조합으로 병렬 처리로 처리
* 데이터 송수신 시 보안을 위해 FHIR 프레임워크 계층 암복호화 인터셉터 구현
* 컨슈머 모듈 Jenkins 빌드 파이프라인, ArgoCD 설정
